# Process Scheduling Simulator

## Project Overview

This project is a **Process Scheduling Simulator** developed in Java to demonstrate how CPU scheduling algorithms work in operating systems.

The program simulates process execution using the **Shortest Remaining Time First (SRTF)** scheduling algorithm, which is the preemptive version of the **Shortest Job First (SJF)** algorithm.

Users enter processes with different **arrival times** and **burst times**, and the simulator determines how the CPU schedules them based on the selected algorithm.

This project was developed as part of a **university Operating Systems course project at King Saud University**.

---

## Features

• Process scheduling simulation  
• Implementation of **Shortest Remaining Time First (Preemptive SJF)**  
• Automatic **Gantt chart generation**  
• Calculation of important performance metrics:

- CPU Utilization  
- Average Turnaround Time  
- Average Waiting Time  

• Simulation of different scheduling scenarios

---

## How the Program Works

1. The user enters:
   - Number of processes  
   - Arrival time for each process  
   - Burst time for each process  

2. The program schedules the processes using the **SRTF algorithm**.

3. The simulator generates a **Gantt Chart** showing the execution order of processes.

4. Performance metrics are calculated, including:

- CPU Utilization  
- Average Waiting Time  
- Average Turnaround Time  

These metrics help evaluate the efficiency of the scheduling algorithm.
